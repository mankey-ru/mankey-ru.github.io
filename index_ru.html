Миграция и архитектура метаданных

Декларативная система для административных форм и пользовательских страниц

Система метаданных определяет два формата для:

  • Административные формы (1000+): Интерфейсы CRUD на основе базы данных
  • Пользовательские страницы (500+): Слой получения данных и композиция компонентов

Эволюция архитектуры:

  • До: Два устаревших формата XML с многочисленными обходными путями и кастомными решениями
  • После: Две элегантные, поддерживаемые структуры JSON с уменьшенными накладными расходами
  • Разработано:
    • Скрипты конвертации для обоих форматов
    • Редакторы форматов для обоих форматов в приложении Electron
    • Приложение админ-панели
Пояснение. Что такое метаданные для административных форм

Наши файлы метаданных — это декларативные JSON-дескрипторы, которые определяют полный жизненный цикл административной формы — от лежащих в основе таблиц базы данных до поведения UI в браузере.

Это устраняет повторяющийся шаблонный код CRUD, сохраняя согласованность для 1000+ форм.

Каждый дескриптор сопоставляет столбцы базы данных (или межтабличные связи) с определениями полей, задающими поведение в трёх контекстах:

  • Представление списка
    • Фильтрация
    • Сам список (отображение, сортировка, редактирование записей на лету)
  • Представление формы (создание, редактирование одной записи)

Каждое поле формы может иметь разные типы элементов управления во всех трёх контекстах, каждый со своими параметрами, такими как валидация, значения по умолчанию или пользовательский шаблон. Примеры типов элементов управления:

  • Поле ввода (текст, целое число, число с плавающей точкой, email и т.д.) или Textarea
  • Выпадающий список (включая асинхронный поиск), Popup Picker, оба могут использоваться для выбора нескольких записей
  • Дата с временем или без, в контексте фильтра может использоваться как диапазон
  • Выбор карты или адреса (геокодер)
  • Редактор диаграмм
  • ...и ещё несколько

Пояснение. Что такое метаданные для пользовательских страниц

Эти файлы метаданных страниц определяют полную HTML-страницу или слой получения данных (на основе JSON), устраняя необходимость в кастомном бэкенд-коде для каждой страницы (всего: 500+ страниц).

Компоненты — это гранулярные строительные блоки, которые:
  • Перечисляют точные поля для выборки
  • Назначают первичные ключи
  • Определяют связи (один-к-одному, один-ко-многим)
  • Обрабатывают пагинацию и фильтрацию
