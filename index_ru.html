  • Разработано:

Объяснение форматов метаданных
Что такое метаданные для админ-форм

Эти файлы метаданных — декларативные JSON-дескрипторы, которые определяют полный жизненный цикл админ-формы — от таблицы (или таблиц) базы данных до поведения в браузере.

Это исключает повторяющийся шаблонный код CRUD, сохраняя согласованность в 1000+ формах.

Каждый дескриптор сопоставляет столбцы базы данных (или межтабличные связи) с определениями полей, задающими поведение в трёх контекстах:

  • Вид списка
    • Фильтрация
    • Сам список (отображение, сортировка, редактирование записей на лету)
  • Вид формы (создание, редактирование одной записи)

Каждое поле формы может иметь разные типы элементов управления во всех трёх контекстах, каждый со своими параметрами, такими как валидация, значения по умолчанию или пользовательский шаблон. Примеры типов элементов:

  • Поле ввода (текст, целое число, дробное, email и т.д.) или текстовая область
  • Выпадающий список (включая асинхронный поиск), всплывающий выбор, оба могут использоваться для множественного выбора
  • Дата с временем или без, в контексте фильтра может использоваться как диапазон
  • Выбор карты или адреса (геокодер)
  • Редактор диаграмм
  • ...и некоторые другие

Что такое метаданные для публичных страниц

Эти файлы метаданных страниц определяют полную HTML-страницу или уровень получения данных (на основе JSON), исключая необходимость в пользовательском бэкенд-коде для каждой страницы (всего 500+ страниц).

Компоненты — это гранулярные строительные блоки, которые:
  • Перечисляют точные поля для выборки
  • Назначают первичные ключи
  • Определяют отношения (один-к-одному, один-ко-многим)
  • Обрабатывают пагинацию и фильтрацию
