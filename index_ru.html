Информаторы версий (IVI + RVI)

Два бэкенд-сервиса: IVI (на стороне компании) и RVI (развёрнут на сервере заказчика — РЖД). Сервисы полностью автоматизируют жизненный цикл выпуска ПО — от принятия запроса на слияние до публикации релиза и установки по цепочке серверных сред (от staging до production).

Ключевые возможности и обязанности:

  • Реализовал структурированное логирование в Elasticsearch с мониторингом в реальном времени.
  • Настроил умные уведомления в Mattermost по нескольким каналам, включая ссылки на формы повторного запуска, логи Kibana и сведения об ошибках.
  • Достиг высокого покрытия тестами с комплексными end-to-end и модульными тестами (Jest).

  • Обработчик публикации релиза: Обнаруживает «потерянные» задачи, загружает и распаковывает дистрибутив, генерирует артефакты для заказчика (6 типов документов) и JSON состава релиза для надёжной передачи данных в RVI (запасной вариант при недоступности REST API).
  • Обработчик событий установки: Фиксирует успешные установки/откаты на сервере, выполняет массовое обновление статусов релизов и связанных задач через Bulker API, добавляет комментарии об откатах.

Пояснение форматов метаданных
Что такое метаданные для форм администрирования

Эти файлы метаданных представляют собой декларативные JSON-дескрипторы, определяющие полный жизненный цикл формы администрирования — от лежащей в основе таблицы(-ц) базы данных до поведения UI в браузере.

Это устраняет повторяющийся CRUD-шаблонный код, сохраняя согласованность на более чем 1000 формах.

Каждый дескриптор сопоставляет столбцы базы данных (или межтабличные связи) с определениями полей, задающими поведение в трёх контекстах:

  • Представление списка
    • Фильтрация
    • Сам список (отображение, сортировка, редактирование записей на лету)
  • Представление формы (создание, редактирование одной записи)

Каждое поле формы может иметь различные типы элементов управления во всех трёх контекстах, каждый со своими параметрами, такими как валидация, значения по умолчанию или пользовательский шаблон. Примеры типов элементов управления:

  • Ввод (текст, целое, дробное, email и т. д.) или текстовая область
  • Выпадающий список (включая асинхронный поиск), всплывающий выбор, оба могут использоваться для выбора нескольких записей
  • Дата с временем или без, в контексте фильтра может использоваться как диапазон
  • Выбор адреса или карты (геокодер)
  • Редактор диаграмм
  • ...и некоторые другие

Что такое метаданные для пользовательских страниц

Эти файлы метаданных страниц определяют полную HTML-страницу или слой получения данных (на основе JSON), устраняя необходимость в пользовательском бэкенд-коде для каждой страницы (всего: 500+ страниц).

Компоненты — это гранулярные строительные блоки, которые:
  • Перечисляют точные поля для выборки
  • Обозначают первичные ключи
  • Определяют связи (один-к-одному, один-ко-многим)
  • Обрабатывают пагинацию и фильтрацию
