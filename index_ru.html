  • Разработано:

    • Редакторы форматов для обоих форматов в Electron-приложении

Объяснение форматов метаданных
Что такое метаданные для Административных форм

Эти файлы метаданных представляют собой декларативные JSON-дескрипторы, которые определяют полный жизненный цикл административной формы - от базовой таблицы (таблиц) базы данных до поведения пользовательского интерфейса в браузере.

Это устраняет повторяющийся шаблонный код CRUD, сохраняя согласованность в 1000+ формах.

Каждый дескриптор сопоставляет столбцы базы данных (или межтабличные связи) с определениями полей, определяющими поведение в трех контекстах:

  • Представление списка
    • Фильтрация
    • Сам список (отображение, сортировка, редактирование записей на лету)
  • Представление формы (создание, редактирование одной записи)

Каждое поле формы может иметь разные типы элементов управления во всех трех контекстах, каждый со своими параметрами, такими как валидация, значения по умолчанию или пользовательский шаблон. Некоторые примеры типов элементов управления:

  • Ввод (текст, целое число, число с плавающей точкой, email и т.д.) или текстовая область
  • Выпадающий список (включая асинхронный поиск), Всплывающий выбор, оба могут использоваться для выбора нескольких записей
  • Дата со временем или без, в контексте фильтра может использоваться как диапазон
  • Выбор карты или адреса (геокодер)
  • Редактор диаграмм
  • ...и некоторые другие

Что такое метаданные для Пользовательских страниц

Эти файлы метаданных страниц определяют полную HTML-страницу или слой получения данных (на основе JSON), устраняя необходимость в пользовательском бэкенд-коде для каждой страницы (всего: 500+ страниц).

Компоненты - это гранулированные строительные блоки, которые:
  • Перечисляют точные поля для выбора
  • Назначают первичные ключи
  • Определяют отношения (один-к-одному, один-ко-многим)
  • Обрабатывают пагинацию и фильтрацию
