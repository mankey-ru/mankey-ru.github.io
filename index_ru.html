Миграция и архитектура метаданных

Декларативная система для административных форм и пользовательских страниц

Система метаданных определяет два формата для:

  • Административных форм (1000+): CRUD-интерфейсы на основе базы данных
  • Пользовательских страниц (500+): Слой получения данных и композиция компонентов

Эволюция архитектуры:

  • До: Два устаревших XML-формата с многочисленными обходными решениями и кастомными решениями
  • После: Две элегантные, поддерживаемые JSON-структуры с уменьшенными накладными расходами
  • Разработано:
    • Скрипты конвертации для обоих форматов
    • Редакторы форматов для обоих форматов в Electron-приложении
    • Приложение панели администратора
Пояснение. Что такое метаданные для административных форм

Наши файлы метаданных — это декларативные JSON-дескрипторы, определяющие полный жизненный цикл административной формы — от таблицы(таблиц) базы данных до поведения UI в браузере.

Это устраняет повторяющийся CRUD-шаблонный код, сохраняя согласованность в 1000+ формах.

Каждый дескриптор сопоставляет столбцы базы данных (или межтабличные связи) с определениями полей, задающими поведение в трех контекстах:

  • Представление списка
    • Фильтрация
    • Сам список (отображение, сортировка, редактирование записей на лету)
  • Представление формы (создание, редактирование одной записи)

Каждое поле формы может иметь разные типы элементов управления во всех трех контекстах, каждый со своими параметрами, такими как валидация, значения по умолчанию или пользовательский шаблон. Примеры типов элементов управления:

  • Поле ввода (текст, целое число, число с плавающей точкой, email и т.д.) или текстовая область
  • Выпадающий список (включая асинхронный поиск), всплывающий выбор, оба могут использоваться для выбора нескольких записей
  • Дата с временем или без, в контексте фильтра может использоваться как диапазон
  • Выбор карты или адреса (геокодер)
  • Редактор диаграмм
  • ...и некоторые другие

Пояснение. Что такое метаданные для пользовательских страниц

Эти файлы метаданных страниц определяют полную HTML-страницу или слой получения данных (на основе JSON), устраняя необходимость в пользовательском бэкенд-коде для каждой страницы (всего: 500+ страниц).

Компоненты — это гранулярные строительные блоки, которые:
  • Перечисляют точные поля для выборки
  • Обозначают первичные ключи
  • Определяют связи (один-к-одному, один-ко-многим)
  • Обрабатывают пагинацию и фильтрацию
